# AI 助手接口规范

**Base URL**：`/api/v1`

> 首期真实后端接入 MiniMax Anthropic-compatible API，模型配置只允许存在于后端环境变量或本地 secret。前端永远只调用 DataGov 后端 AI Gateway。

## 10.1 获取 AI 能力

- **Endpoint**：`GET /api/v1/ai/capabilities`
- **Auth**：`Authorization: Bearer <token>`
- **Response `data`**：

```json
[
  {
    "id": "write-sql",
    "title": "写 SQL",
    "description": "按业务口径生成 SQL",
    "prompt": "帮我写一段 PostgreSQL SQL...",
    "icon": "sql",
    "accent": "cyan"
  }
]
```

## 10.2 会话管理

### `GET /api/v1/ai/conversations`

查询当前用户 AI 会话列表。

Query：

- `search`：可选，按标题搜索。
- `includeArchived`：可选，是否包含归档会话。
- `limit`：可选，默认 30。

### `POST /api/v1/ai/conversations`

创建会话。

```json
{
  "title": "订单明细 SQL 分析",
  "context": {
    "viewId": "script-dev",
    "viewTitle": "脚本开发",
    "url": "http://127.0.0.1:5174/?view=script-dev",
    "dialect": "PostgreSQL"
  }
}
```

### `GET /api/v1/ai/conversations/{id}`

返回会话详情和消息历史。

### `PATCH /api/v1/ai/conversations/{id}`

更新标题、收藏、归档状态。

```json
{
  "favorite": true,
  "archived": false,
  "title": "新的会话标题"
}
```

## 10.3 发送消息

### 兼容接口

`POST /api/v1/ai/assistant/messages` 保留为兼容入口。后端会自动创建会话并返回同样的 AI 回复结构。

## 10.4 上下文、偏好、反馈与工具

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `POST` | `/api/v1/ai/context/preview` | 预览本次请求上下文、Token 估算和脱敏命中 |
| `GET` | `/api/v1/ai/preferences` | 读取当前用户 AI 偏好 |
| `PUT` | `/api/v1/ai/preferences` | 更新回答风格、SQL 方言、记忆开关 |
| `POST` | `/api/v1/ai/messages/{id}/feedback` | 提交点赞、点踩或中性反馈 |
| `POST` | `/api/v1/ai/messages/{id}/regenerate` | 基于同一问题重新生成 |
| `POST` | `/api/v1/ai/behavior-events` | 记录复制、采纳、建议点击、重新生成等行为 |
| `GET` | `/api/v1/ai/token-usage` | 查询最近 30 天 Token 用量与配额 |
| `GET` | `/api/v1/ai/tools` | 查询可用只读工具 |

## 10.5 后端处理要求

- **Provider**：首期使用 MiniMax Anthropic-compatible API，模型 `MiniMax-M3`。
- **安全**：入模前脱敏连接串、密码、token、Authorization 等敏感信息。
- **权限**：所有会话、消息、反馈、偏好按当前登录用户隔离。
- **Token**：每次请求记录输入、输出、总 Token、模型、延迟和上下文预算。
- **工具**：首期只读工具优先，不直接访问外部源数据，外部源数据长期走独立 Connector Gateway / CDM Proxy。
- **Fallback**：默认 MSW passthrough 到 Go API；如需兜底演示，设置 `VITE_REAL_AI_ASSISTANT=false`。
