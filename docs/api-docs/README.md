# DataGov API 文档指南

欢迎查阅数据治理系统（DataGov）的 API 接口文档。

为了便于前后端联调、功能验证以及后期扩展，本目录下的文档已按照**业务模块**进行了结构化拆分，分为“页面与接口映射”和“接口详细规范”两大部分。

## 目录结构

### 1. [页面与接口映射 (Mapping)](./mapping/README.md)
该目录整理了前端系统中所有页面及其所依赖的 API 接口，并说明了这些接口对应的页面功能和所在菜单路径。
- 适用于**前端开发人员**快速查找某个页面需要调用哪些接口。
- 适用于**测试人员**根据页面功能定位到具体的后端路由。

### 2. [接口详细规范 (Specification)](./specification/README.md)
该目录提供了与后端开发对接的契约草案，详细定义了每个 API 的请求路径（Endpoint）、方法（Method）、请求参数（Request Schema）以及返回的数据结构（Response Schema）。
- 适用于**后端开发人员**作为接口设计和数据结构返回的参考标准。
- 适用于**联调阶段**的数据校验。

## 业务模块概览

两个子目录均按照以下 9 个核心业务模块进行了一一对应的拆分（以 `01` 到 `09` 编号）：

1. **认证鉴权 (Auth)**: 登录、注册、密码找回等基础认证功能。
2. **工作台 (Dashboard)**: 综合仪表盘、数据治理看板的核心统计与趋势数据。
3. **数据资产 (Asset)**: 资产总览、资产注册、数据目录、数据地图、数据血缘及数据源管理。
4. **元数据管理 (Metadata)**: 元数据采集、查询、维护及模型管理。
5. **数据标准 (Standard)**: 标准定义、映射、评估，以及数据字典和码值管理。
6. **数据开发 (Development)**: 数据同步、脚本开发、实时计算与任务编排。
7. **数据质量与监控 (Quality)**: 业务/技术指标管理及系统运维监控告警。
8. **数据服务 (Service)**: API 服务发布管理及数据共享授权流转。
9. **系统管理 (System)**: 用户、角色权限管理，以及操作日志与全局配置。

## 维护规范

- **新增接口**：请务必同时更新 `mapping` 中的页面依赖说明以及 `specification` 中的接口定义。
- **修改接口结构**：请在对应的 `specification/xx-module.md` 中更新 JSON Schema，并通知相关的前端/后端开发人员。
- **保持编号一致**：新增模块时，请在两个子目录中创建相同编号（如 `10-xxx.md`）的文件，以保持结构的统一性。
